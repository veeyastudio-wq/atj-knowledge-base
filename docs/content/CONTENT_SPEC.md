# ATJ content programme — structure and frontmatter spec

Every article in the content programme must carry the following frontmatter fields. No article is published without all fields populated.

title: ""

description: ""

author: "Vilam Vuong"

author_bio: "Vilam navigated the UK family court system himself. Access to Justice was built from that experience."

reviewer_name: ""

reviewer_sra: ""

published_date: ""

reviewed_date: ""

cluster: ""

pillar: true/false

slug: ""

Fields:
- title: the exact question or statement the article answers
- description: one sentence, plain English, what the reader will understand after reading
- author: always "Vilam Vuong"
- author_bio: short biography, to be finalised before first publish
- reviewer_name: full name of the solicitor who reviewed this article
- reviewer_sra: the reviewer's SRA number
- published_date: YYYY-MM-DD format
- reviewed_date: YYYY-MM-DD format, must match or follow published_date
- cluster: one of "child-arrangements", "financial-remedy", "divorce"
- pillar: true if this is the pillar page for the cluster, false for all cluster articles
- slug: the URL path for this article, lowercase, hyphens only
