# ATJ Retrieval Evaluation

This directory contains the golden set and evaluation outputs for the ATJ embedding pipeline sign-off.

## Files

| File | What it is |
|---|---|
| `golden_set.json` | LLM-generated query-chunk pairs used as ground truth. Generated by `scripts/generate_golden_set.py`. |
| `golden_set_summary.txt` | Summary of the golden set generation run. |
| `retrieval_results.json` | Full per-query results from the most recent evaluation run. |
| `retrieval_report.txt` | Human-readable summary with overall pass/fail. |

## How to run

### Step 1 — Generate the golden set

```bash
python scripts/generate_golden_set.py
```

Samples chunks from pgvector, calls Claude to generate realistic user questions for each,
outputs `eval/golden_set.json`. Takes ~5 minutes. Requires `ANTHROPIC_API_KEY` in `.env`.

### Step 2 — Run the evaluation

```bash
python scripts/evaluate_retrieval.py
```

Runs every query through hybrid retrieval, checks whether the expected chunk appears
in the top 10 results, and outputs a pass/fail report.

### Step 3 — Check the report

```bash
cat eval/retrieval_report.txt
```

### Optional — test a single query interactively

```bash
python scripts/retrieve.py "What happens at a First Appointment hearing?"
python scripts/retrieve.py "What is Form E?" --layer layer2 --top-k 5
python scripts/retrieve.py "Can I bring someone to court with me?" --json
```

## Pass threshold

Context recall @ 10: **75% or above** = pipeline signed off.

If the evaluation fails, read the missed queries in `retrieval_report.txt` first.
Many failures are fixable by reviewing whether the question genuinely matches
the expected chunk — not by changing retrieval parameters.

## Environment

Docker container `atj-db` must be running:

```bash
docker start atj-db
```

`.env` at repo root must contain:
```
OPENAI_API_KEY=...
ANTHROPIC_API_KEY=...
```
